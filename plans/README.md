## Findings considered and rejected

- **Branch: egg edit/delete removal**: Submitted as Plan 010 (highest priority).

- **SQL injection in `ensureDatabase` via string interpolation** (`db.ts:34`): The `DB_NAME` value comes from a Docker env var, not user input. Code-hygiene finding, but the attack surface is the operator's own env vars. Not worth the plan — the risk is negligible for a LAN-local app.
- **`getUserByEmail` called twice per auth flow** (`auth.ts:23,31`): Negligible performance impact for a small-flock app with ~5 users. Not worth a plan.
- **Hardcoded Southern Hemisphere season mapping** (`analytics.ts:87-92`): Documented design choice per `CONTEXT.md`. Not a bug.
- **All pages client-side rendered** (`src/app/*/page.tsx` `"use client"`): ADR 0005 documents the always-online-over-LAN design, making SSR unnecessary. Could still be optimized but is by-design, not a finding.
- **CSRF not mitigated**: Risk is real but the app is LAN-local with a trusted user base. Mitigation would add complexity with limited practical benefit in this deployment context. Deferred.
- **No rate limiting**: Same reasoning as CSRF — LAN-local, small trusted user base, not worth the middleware complexity.
